# Tutoring loop

## The product is the loop

Conversation identifies the current learning need. A lesson creates a deliberate
experience and structured evidence. The return makes that evidence visible to
the tutor. Conversation then interprets it and chooses what should happen next.

```text
mission -> diagnose -> research -> lesson -> return -> discuss -> update -> act
```

## Next-best-action policy

Use a lesson only when it adds value beyond chat. Rank possible actions by:

1. relevance to the mission
2. expected learning value now
3. fit to current capability and assistance history
4. learner energy, time, and stated desire
5. evidence quality the action can produce
6. cost and complexity

The smallest effective action wins.

## Milestones

Milestones are optional tangible capabilities, not chapter numbers. They may be
split, merged, reordered, deferred, or abandoned as evidence changes.

Recommended statuses:

- `unknown`
- `introduced`
- `practiced_with_support`
- `independent`
- `transfer_demonstrated`
- `needs_revisit`

A milestone is not complete because a lesson covered it. Completion requires
observable evidence, preferably independent application or transfer.

## Adaptivity at two scales

High level:

- goals, outcomes, constraints, motivation
- rough capability map
- meaningful projects or demonstrations

Micro level:

- the exact misconception surfaced today
- which representation helped
- where hints were required
- confidence and frustration
- open questions
- what should be retried, contrasted, or explained next

The next lesson should visibly respond to the micro evidence while still moving
toward a high-level outcome.
